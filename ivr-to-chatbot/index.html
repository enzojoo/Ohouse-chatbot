CS 응대 시나리오 전환 : IVR → 인서비스 챗봇

🔑 핵심 변경 포인트

  • 진입 메뉴 4 → 3 단순화. 4번 기타 항목을 없애고, 기타성 문의는 챗봇 자유발화로 흡수해 의도를 판단합니다.
  • 1번이 "고정 챗봇"에서 "챗봇 우선 + 핸드오버"로. 챗봇이 끝까지 처리하려 시도하고, 해결 불가 케이스만 상담원에게 넘깁니다.
  • 2·3번은 즉시 상담원 연결 유지. 시공·콘텐츠는 챗봇 자동응대 대상이 아니므로 기존 정책 그대로 사람에게.
  • 번호 선택에 갇히지 않음. 메뉴를 안 고르고 바로 질문해도 챗봇이 1/2/3 의도로 분류해 응대 또는 이관합니다.

"오늘의집 AI 상담원 오집사였습니다. 좋은 하루 보내세요. 상담이 종료되었습니다."
🔁 챗봇 → 상담원 핸드오버 트리거 6종 운영시간 내
  1. 고객 직접 상담원 요청
    "필요하시면 상담원 연결도 도와드릴까요?"
  2. 반복 문의 / 미해결 감지
    "불편을 드려 죄송합니다. 해당 내용은 우선 확인이 필요해 상담원에게 바로 연결해 드리겠습니다."
  3. 챗봇 처리 불가 실행 요청
    "상담원 연결을 도와드릴게요. 잠시만 기다려 주세요."
  4. KMS 안내가능 절차 문의
    "해당 요청은 상담원이 빠르게 확인할 수 있도록 바로 연결해 드릴게요."
  5. 고객 감정 / 불만 감지
    "제가 충분히 도움을 드리지 못하고 있는 것 같습니다. 문의 해결을 위해 상담원에게 연결해 드리겠습니다."
  6. 챗봇 응답 불가 (저신뢰도)
    "해당 요청은 상담원이 가장 정확하게 도와드릴 수 있어요. 상담원 연결을 도와드릴까요?"
⏱ 무응답 1차(5분)→2차(+2분)→3차(+3분) 종료 · ⚠️ 운영시간 외엔 상담원 인계 불가 안내 후 운영시간 재문의 유도

🕘 논의사항 ② — 챗봇 도입 후 상담 운영시간 조정

현재는 평일 09:00~18:00만 상담 운영 중. 인서비스 챗봇이 들어오면 운영시간을 24시간 대응으로 확대할지, 아니면 클레임 API 연동 완료 후 24시간 대응으로 갈지 결정이 필요합니다. 아래 2개 시나리오로 검토합니다.

A
기존(09~18) 유지
▸ 단기 / 리스크 낮음
  • 운영 정책 변경 없음 — 특이사항 없음
  • 야간/주말은 챗봇 자동응대 + 운영시간 외 안내로 처리 (상담원 인계는 운영시간 재문의 유도)
  • 이후 클레임 API 오픈 시점에 24시간 운영 재검토 진행
B
24시간 운영으로 변경
▸ 확대 / 사전 준비 필요
  • 업무시간 외 상담원 처리 필요건 발생 시, 1:1 게시판 작성 등으로 유도할 수 있는 제반 기능 확보 필요
  • 야간 발생 문의의 순차 처리를 위해 상담원 추가 투입 필요 가능성 존재
⚠️ 인력·기능 준비가 선행되어야 함 (단순 시간 확대 아님)
결정 포인트 — ①번 챗봇이 24시간 자동응대를 하더라도, "상담원이 필요한 건"을 야간에 어떻게 처리할지가 핵심. 챗봇 자동응대 범위만 24시간 열고 상담원 인계는 운영시간 유지(A)로 시작 → 클레임 API·인력 확보 후 전면 24시간(B)으로 단계 전환하는 안도 가능.
