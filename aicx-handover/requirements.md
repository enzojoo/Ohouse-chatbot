# AICX 어드민 — 상담원 실시간 응대(핸드오버) 기능 요구사항 v2.1

> 작성자: 서비스운영팀 / 시나리오 담당
> 대상 독자: AICX 개발팀, 내부 TPM·PO·Eng
> 본 문서는 ECA(현 사용 중인 상담 시스템)의 채팅상담 기능을 AICX 어드민에 이식하기 위한 기능·기술 요구사항입니다.
> 시각 명세는 `index.html`(31개 화면 디테일 — S01~S31, 보강 J 그룹 포함)을 참고하세요.

## 1. 배경

- 현재 카카오톡 채널로 인입되는 상담은 ECA 어드민에서 상담원이 처리하고 있음.
- 챗봇(오집사)을 인앱/웹으로 내재화하면서, **챗봇 SDK가 인계한 상담을 AICX 어드민 내부에서 상담원이 실시간으로 받아 처리**해야 함.
- 현재 AICX 어드민은 "챗봇 대화 모니터링/종료 티켓 조회"까지만 지원하며, 상담원 실시간 채팅 기능이 부재.
- 카카오 등 외부 채널 의존을 제거하고 인앱/웹 통합 운영을 목표로 함.

## 2. 범위

### In scope
- 챗봇 → 상담원 핸드오버 트리거 → **스킬별 큐 적재 → 자동 매칭(Push)** 〔v2.1〕
- 상담원 멀티 채팅 동시 응대(관리자 설정 가변)
- 실시간 메시지 송수신 (텍스트·이미지·파일·시스템 메시지)
- 챗봇 컨텍스트(인텐트·신뢰도·감정·검증된 사용자 정보·AI 요약) 인계
- 상담사 간 **티켓 이관**(실시간 이관 폐지) 〔v2.1〕
- 관리자 지원상담(3자 개입) + **실시간 감청** 모드 〔v2.1〕
- 추천 답변/매크로/단축키/금칙어 마스킹
- 티켓 자동 발급·작성·종료 (주문정보 자동 연동) 〔v2.1〕
- 고객 정보 / 주문 / 인계 타임라인 사이드 패널
- **매니저 실시간 모니터링 화면** (전체 상담사 + 큐 + 임계치 알림) 〔v2.1〕
- **관리자 통계 대시보드** (실시간/일/주/월) 〔v2.1〕
- **관리자 설정 메뉴** (스킬·슬롯·우선순위·금칙어·마스킹·SLA) 〔v2.1〕
- **인앱 NPS** (스마일/엄지 단일 액션) 〔v2.1〕
- **영업시간 외 챗봇 단독 응대 분기** 〔v2.1〕

### Out of scope (이번 페이즈 미포함)
- 전화·메일 채널 통합 (이번 페이즈는 인앱/웹 채팅만)
- SMS 발송 / 첨부파일 링크 발송 (ECA의 SMS 기반 기능)
- 비회원 매칭 / 주문 추출 (인앱/웹 SDK는 유저ID 기반으로 항상 식별됨)
- CTI 미니전광판 / 소프트폰
- 핸드오버 거절/재배정 정책 〔v2.1 제거 — Push 자동 매칭으로 단순화〕
- 실시간 이관(호전환) 〔v2.1 제거 — 티켓 이관으로 일원화〕
- 다수 계정 분기 케이스 〔v2.1 제거 — 내재화 오집사 로그인 세션 기반으로 불필요〕
- 카카오톡 채널 별도 운영 〔v2.1 제거 — 채널 일몰 예정〕

## 4. 화면별 기능 요구사항

### 4.1 상단 헤더 영역
- **CTI 상태 토글**: `대기` / `이석` / `OB` / `로그아웃`. 상태 전환 시 인입 수신 여부 즉시 반영.
- **응대 카운터**: 현재 응대 중 / 최대 동시 응대 수 / 미수락 건수 노출.
- **알림 센터**: SLA 임박, 매니저 멘션, 이관 요청 수신, 챗봇 오분류 신고 답신 등.
- **SLA 임박 배너**: 잔여 시간 1분 이하 건이 있을 때 상단 강조 노출.

### 4.2 좌측 채팅 리스트
- **탭(처리상태)**: 진행중 / 대기 / 종료 / 전체. 챗봇 단독 종료 건은 별도 라벨(`챗봇 종료`).
- **검색**: 고객명 / 메시지 본문 / 주문번호 / 태그(`#`).
- **카드 정보**: 고객명, 채널 출처(앱/웹), 처리상태 배지, 최근 메시지 1줄, 주문번호 또는 인텐트 라벨, 상대 시각.
- **무응답 인디케이터**: 고객/상담사 60초 이상 무응답 시 카드에 펄스(초록) 표시 — ECA 동일 사양.
- **SLA 카운트다운**: 잔여 시간 표시(예: `잔여 1:12`), 1분 미만은 빨강.
- **알림음**: 신규 인입 / 무응답 60초 도달 / SLA 임박 — 각 다른 톤.

### 4.3 가운데 대화 영역
- **헤더 메타 라인 1**: 고객명, 채널, 회원 등급, 주문번호.
- **헤더 메타 라인 2**: 상태 / 우선순위 / 그룹 / 유형 / 메모(자유입력 ≠ enum), 태그.
- **시스템 메시지**: 챗봇 시작·인계·종료·매니저 참여·이관·SLA 도달 등은 시스템 라인으로 표시(말풍선 아님).
- **말풍선 시각 구분**:
  - 고객(좌, 흰색 + 보더)
  - 챗봇(우, 라이트 블루)
  - 상담사(우, 메인 블루)
  - 매니저/지원상담(우, 앰버)
- **AI 핸드오버 요약 카드**: 인계 시점에 인라인으로 박힘. 분류 인텐트(Top3) / 신뢰도 / 감정 / 사전 검증된 정보 / 자연어 요약. 응대 시작 전 필독 영역.
- **금칙어 마스킹**: 양방향(고객→상담사, 상담사→고객) `*` 처리.
- **파일·이미지 첨부**: 인라인 미리보기, 다운로드/저장 가능.
- **추천 답변**: 입력창 위 칩으로 KMS·템플릿·주문 상태 기반 3~5개 자동 노출.

### 4.4 하단 입력창(컴포저)
- **/ 매크로 검색**: `/` 입력 시 추천메시지·스니펫 자동완성.
- **@ 매니저 호출**: `@매니저명` 입력 시 해당 매니저에게 지원상담 알림.
- **툴바**: 사진 첨부, 이미지 링크 발송, 템플릿, 고객정보 입력 요청, 티켓 이동, 매니저 호출, 상담 종료 안내, 종료, 이관, 전송.
- **단축키**: `⌘+Enter` 전송 / `Alt+0~9` 매크로 슬롯 / `⌘+K` 빠른 검색 / `Esc` 인계 거절.

### 4.5 우측 사이드 패널 (탭)
- **고객 정보**: 유저ID, 휴대폰, 이메일, 채널(앱/웹·디바이스·앱 버전), 최근 접속.
- **상담 현황**: 상태, 담당자, 대기시간, SLA 잔여, 최근 업데이트.
- **연결된 주문**: 다건 가능. 카드 단위로 주문번호·상품·금액·상태·배송상태. 추가 연결 액션.
- **인계/이관 타임라인**: 챗봇 시작 → 상담원 인계 → 매니저 참여 → 이관 등 이벤트를 시간 순서로.
- **이전 이력**: 최근 N주 티켓, 채널 전체 통합(웹/앱). 더블클릭으로 과거 대화 팝업.
- **태그**: 추가/삭제. 헤더 라인의 태그와 단일 소스.

### 4.6 인입 알림 모달 / 토스트
- 우측 하단 토스트로 노출, 잔여 시간 시각화(원형 카운트다운).
- **15초 이내 미응답 시 자동 거절** → 큐의 다음 상담원에게 재배정.
- 토스트 내용: 고객명, 채널, 한 줄 본문, 인텐트 태그, 사전 인증 여부, 핸드오버 신뢰도.

### 4.7 종료 후 티켓 작성
- 상담 종료 시 티켓 자동 생성(이미 인입 시점에 발급되어 있음).
- 종료 직후 모달에서 **상담구분 / 상담유형 / 연결 주문번호 / 상담 내용 요약(AI 초안 제공) / 태그** 확정.
- 저장 전 종료 불가(미저장 종료 방지).

## 5. 챗봇 ↔ 어드민 핸드오버 프로토콜

### 5.1 트리거 조건 (봇 측)
- 인텐트 신뢰도 < 임계값
- 응답 불가 영역(주문조회/취소접수/반품·교환 등) 명시 인텐트
- 강성 키워드 / 부정 감정 강도 임계 초과
- 동일 인텐트 N회 반복(루프 감지)
- 고객 명시 요청("상담사 연결")

### 5.2 핸드오버 페이로드 (봇 → 어드민)
```
{
  "conversation_id": "...",
  "channel": "app_ios" | "app_android" | "web",
  "user": { "id": "...", "is_authenticated": true, "tier": "prime" },
  "verified_context": {
    "order_ids": ["OH-..."],
    "auth_method": "session",
    "device": "iOS 26.11"
  },
  "intent": {
    "top": [{ "name": "shipping_delay", "confidence": 0.86 }, ...],
    "category": "logistics"
  },
  "sentiment": { "label": "anxious", "intensity": 0.6 },
  "handover_reason": "out_of_bot_scope",
  "ai_summary": "고객은 토요일 배송 약속 여부를 ...",
  "history": [ /* 전체 봇 대화 turn */ ],
  "sla_target_seconds": 120,
  "priority": "high" | "normal" | "low",
  "tags": ["#배송조회","#대형가구"]
}
```

### 5.3 어드민 → 봇 (역방향 이벤트)
- 상담원 응대 시작 / 종료 → 봇은 해당 대화 재진입 차단 또는 폴백 활성화
- 상담원이 "봇으로 환원" 선택 시 → 봇 복귀(드물지만 필요)
- 오분류 신고: 인텐트 카테고리/사유를 봇 학습 데이터로 회송

### 5.4 실시간 전송 (양방향)
- WebSocket(또는 동등 수단) 기반 양방향 채널 1개.
- 메시지 타입: `user_message` / `bot_message` / `agent_message` / `manager_message` / `system_message` / `typing` / `read` / `attachment`.
- 메시지 단위 idempotency-key 권장(중복 방지).
- 클라이언트 재연결 시 마지막 수신 메시지 ID로 누락분 backfill.

## 6. 상담원 상태 모델 〔v2.1 개정〕

| 상태 | 인입 수신 | 큐 매칭 | 전환 방식 | 비고 |
|---|---|---|---|---|
| **대기 (Ready)** | O | 자동 매칭 대상 | 수동 | 본인 스킬 큐 중 가장 오래 대기한 고객과 자동 연결 |
| **응대중 (Busy)** | 슬롯 여유 시 O | 슬롯 한도 내 매칭 | 자동 (매칭 시) | 슬롯 가득 시 자동 "수용불가" |
| **후처리 (Wrap-up)** | X | 큐 제외 | 자동 (응대 종료 시) | SLA 초과 시 매니저 알림. **자동 복귀 없음** — 수동으로 "대기" 전환 필요 |
| **이석 (Away)** | X | 큐 제외 | 수동 | 자동 대기 전환 시간 설정 없음 |
| **회의 (Meeting)** | X | 큐 제외 | 수동 | |
| **교육 (Training)** | X | 큐 제외 | 수동 | |
| **점심 (Lunch)** | X | 큐 제외 | 수동 | |

- 동시 응대 한도(슬롯)는 관리자 설정값. 스킬별 기본값 설정 가능(예: 커머스 5건, 컨텐츠 3건, O2O 3건).
- **수락/거절 단계 없음**: "대기" 전환만으로 큐 매칭 대상. Push 방식.
- **다중 스킬 보유자**: 본인이 보유한 스킬 큐 중 가장 오래 대기한 큐의 고객 우선 매칭.
- **스킬 변경 권한**: 관리자만(어드민 설정). 본인 변경 불가.

## 8. 비기능 요구사항

- **지연**: 메시지 송수신 P50 < 500ms / P95 < 1.5s
- **가용성**: 99.9% (운영시간 기준)
- **동시 응대**: 상담사 1인당 동시 5건 기본, 운영 설정 가능
- **메시지 보존**: 최소 2년 (KMS 정책 준수)
- **감사 로그**: 상담사 발화·이관·강제종료·매니저 개입 전수 기록
- **개인정보**: 마스킹 정책에 따라 응대 종료 후 일부 필드 자동 마스킹

## 10. 챗봇 어드민이 모니터링 화면으로 가져야 할 추가 사양 (현 AICX 보강 요청)

현재 AICX 채팅 내역 화면에서 다음 항목을 보강해주세요(이미 1차 피드백으로 전달).

- 봇 이름 정합성(`집들이봇-AI` vs `오집사` 통일)
- 핸드오버 사유·인텐트·신뢰도를 시스템 메시지에 명시
- AI 요약 박스의 위치를 인계 시점 인라인 또는 우측 패널 최상단으로
- 챗봇/상담사 말풍선 색·아이콘 분리
- 좌측 리스트 라벨에서 처리상태와 문의유형 축 분리
- 태그를 헤더와 우측 패널에 중복 노출하지 않기 (single source)
- 멀티 주문 케이스를 위한 주문 다건 연결 UI
- 메모 필드의 자유입력/enum 정책 정의
- 알림 3건의 알림 종류 정책(SLA 임박/이관 수신/미응대 등) 정의

## 12. 추후 정의 필요 (mockup에는 기능 슬롯만 제공)

2. **이관 처리 SLA 수치** — 이관 받은 티켓 N시간 내 처리 — 운영팀 협의
3. **임계치 알림 기준 수치** — 후처리 N분 / 대기 X건 / FRT Y초 — 운영 1개월 데이터 후 결정
4. **티켓 자동 채움 범위** — 어디까지 AI가 채우고 어디부터 상담사 수기인지 — AICX 협의
5. **ECA 필드 매핑 표준** — type_cd / contact_cd / 처리결과 코드 등 - AICX·서비스운영 협의
6. **AI 초안 학습 반영 방식** — 상담사 수정값을 어떻게 차기 응대에 반영할지 - AICX 협의
7. **챗봇 자체 종결 시 주문정보 연동** — 5/14 슬랙 논의의 3단 작업(DB 필드 추가 / 중계서버 전달 / ECS payload 추가) 일정 협의 필요
8. **인앱 NPS UI 옵션 선택** — A(스마일 3단계) / B(엄지 업·다운) / C 중 1차 선택 (권장: A)
9. **NPS 인앱 ↔ ECA 데이터 매핑** — 인앱 응답값을 ECA 30분 후 발송 흐름과 어떻게 연동할지
10. **영업시간 외 강성 키워드 케이스** — 다음 영업일 최우선 OB 큐 자동 적재 정책 (선택)
